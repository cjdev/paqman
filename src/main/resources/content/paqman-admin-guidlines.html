Guidelines for program administrators

Step 1: Understand the paqman philosophy

paqman is designed to facilitate the rapid roll-out and maintainance of practical know-how across an organization. It favors:
  • human judgement over machine grading
  • practical demonstrations over reading, tests & course-work
In order for a qualification program to be successful:
  • It should have a well defined, specific focus
  • This focus should be the aquisition of a demonstratable capability
  • The program should be small enough to be maintainable for the administrator
  • The administrator's maintainance cost should be small ... less than the time they would otherwise take to train 2 people

Step 2: Establish The Scope of Your Qualification Program

A qualification program should be focused on one particular skill or capability, where that skill or capability can be detected by means of a brief "practical" examination. The first step in establishing the focus of your program is to choose a name and a description:
  1. name: Identify the skillset/capability that the course is focused on. Then, find a short, memorable description of the kind of person that has this capability.
  2. description: Identify the contexts in which you need to track the acquisition and maintainence of this capability. It should fit in the following form: "you should have this if you _________"
Example:
"break-room barrista": you should have this is if you operate the industrial coffee machines in the break room

Step 3: Build the first version of the program

A qualification program consists of one or more hunks. There are 2 types of hunks:
  • challenge A valid program must have at least one 'challenge' hunk. A challenge consists of a description of how one can go about prooving that they know/are-able-to-do one or more aspects of the overall program capability
  • information An informational hunk does not require any action on behalf of the challenger; it is provided for informational purposes only, and may or may not need to be read/seen/heard by a given challenger.
Tips:
  • Break apart larger programs into multiple challenges. Having more than one challenge has the following benefits:
    • When you make edits, there will be less thrash for those who have already completed the program, as they will only have to re-take the challenges that have changed.
    • It will be easier for first-time challengers, as they will be able to tackle the program in more manageable chunks
  • Favor videos over written material ... it's easier to consume, more like face-to-face communication, etc.

Step 4: Maintain the program
